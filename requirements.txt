Практична робота
Тема: Основи CSS. Оформлення багатосторінкового HTML-сайту
1. Мета роботи
Закріпити практичні навички підключення зовнішніх таблиць стилів і оформлення багатосторінкового HTML-сайту засобами CSS: налаштування кольорової палітри, типографіки, відступів, блокової моделі, оформлення навігації, списків, зображень, таблиць, форм, мультимедійних елементів та базової адаптації під різні розміри екрана.
2. Очікувані результати навчання
• підключати зовнішній CSS-файл до всіх сторінок сайту та організовувати стилі у зрозумілій структурі;
• використовувати селектори елементів, класів та ідентифікаторів, а також групування селекторів;
• застосовувати базові принципи каскаду, спадкування та блочної моделі CSS;
• оформлювати текстовий контент: шрифти, розміри, кольори, міжрядкові інтервали, відступи та акценти;
• стилізувати меню навігації, кнопки, списки, зображення, таблиці, аудіо, відео та HTML-форму;
• використовувати Flexbox щонайменше у двох частинах інтерфейсу;
• створювати базову адаптацію інтерфейсу за допомогою media queries;
• перевіряти візуальний результат у браузері та усувати очевидні проблеми верстки.
3. Необхідне ПЗ та підготовка
• Редактор коду: Visual Studio Code або інший редактор із підсвіткою HTML/CSS.
• Браузер: Google Chrome, Mozilla Firefox, Microsoft Edge або інший сучасний браузер.
• Наявний виконаний проєкт із практичної роботи №1.
• Рекомендовано використовувати Live Server або інший локальний сервер для швидкого перегляду змін.
4. Вихідні умови та структура проєкту
За основу необхідно взяти мінісайт, створений у попередній практичній роботі. Після виконання CSS-оформлення проєкт має мати щонайменше таку структуру:
lab02-css/
├─ index.html
├─ pages/
│  ├─ about.html
│  └─ contact.html
├─ assets/
│  ├─ img/
│  └─ media/
├─ styles/
│  ├─ style.css
│  └─ responsive.css
└─ README.md
	

Обов'язкова умова: Усі три HTML-сторінки повинні використовувати спільний дизайн. Дозволяється додавати класи та ідентифікатори до вже створеної HTML-розмітки, але не можна спрощувати попередню структуру сайту або видаляти обов'язкові елементи з практичної роботи №1.
	

5. Завдання
5.1. Підключення стилів і базова організація CSS
• Створіть папку styles та файли style.css і responsive.css.
• Підключіть style.css на всіх сторінках сайту. Файл responsive.css підключіть після основного, щоб його правила могли перевизначати базові стилі.
• Для index.html використайте відносний шлях ./styles/style.css, а для сторінок у папці pages — ../styles/style.css.
• За потреби додайте до HTML елементів класи container, section, card, site-header, site-footer, nav-list, form-group тощо.
• Усі стилі мають бути винесені в окремі CSS-файли. Вбудовані стилі через style="..." не використовувати.
<!-- index.html -->
<link rel="stylesheet" href="./styles/style.css">
<link rel="stylesheet" href="./styles/responsive.css">

<!-- pages/about.html -->
<link rel="stylesheet" href="../styles/style.css">
<link rel="stylesheet" href="../styles/responsive.css">
	

5.2. Базові глобальні стилі та дизайн-система
• Задайте для всіх елементів box-sizing: border-box.
• Для body приберіть стандартний margin, задайте шрифт, колір тексту, колір тла та line-height.
• Створіть базову палітру сайту: основний колір, акцентний колір, колір тла, колір тексту, колір для меж/ліній.
• Додайте клас .container для обмеження максимальної ширини контенту та центрування сторінки.
• Створіть єдиний візуальний стиль для section або інших тематичних блоків: внутрішні відступи, зовнішні відступи, фон, заокруглення або тінь.
:root {
 --color-primary: #1d4e89;
 --color-accent: #ffb703;
 --color-bg: #f7f9fc;
 --color-surface: #ffffff;
 --color-text: #1f2937;
 --color-border: #d9e2ec;
}

*,
*::before,
*::after {
 box-sizing: border-box;
}

body {
 margin: 0;
 font-family: Arial, sans-serif;
 line-height: 1.6;
 color: var(--color-text);
 background-color: var(--color-bg);
}

.container {
 width: min(1100px, calc(100% - 32px));
 margin: 0 auto;
}
	

5.3. Оформлення header, навігації та footer
• Оформіть шапку сайту так, щоб вона візуально відокремлювалася від основного контенту.
• Меню навігації реалізуйте горизонтально за допомогою Flexbox.
• Для посилань меню задайте стилі normal, hover та active/current (активну сторінку можна позначити класом .active).
• Додайте відступи між пунктами меню, змініть колір тексту, приберіть стандартне підкреслення та створіть зрозумілий стан наведення.
• Оформіть footer у тому ж стилі, що й header: інший фон, вирівнювання тексту, дрібніший розмір шрифту.
5.4. Оформлення головної сторінки index.html
• Стилізуйте головний заголовок, підзаголовки та абзаци. Забезпечте логічну ієрархію розмірів.
• Оформіть списки так, щоб вони були читабельними: відступи, міжрядкові інтервали, маркери або нумерація.
• Зображення та figure повинні виглядати акуратно: адаптивна ширина, заокруглення, тінь або рамка.
• Таблиця має мати підпис, відступи в комірках, рамки, контрастний заголовок і чергування кольору рядків.
• Одна з ділянок сторінки повинна бути оформлена як картка або інформаційний блок із фоном та внутрішніми відступами.
5.5. Оформлення сторінки about.html
• Оформіть текстову частину сторінки у вигляді окремого контентного блоку.
• Відео й аудіо розмістіть у візуально відокремлених секціях або картках.
• Задайте для video ширину 100% із розумним max-width, щоб медіа не виходило за межі контейнера.
• Використайте Flexbox щонайменше в одному блоці сторінки «Про мене»: наприклад, текст + медіа, або дві медіакартки поруч.
• Підписи до медіа мають бути стилізовані та узгоджені з загальним дизайном.
5.6. Оформлення сторінки contact.html
• Форма має бути розміщена в окремій картці або секції з фоном, рамкою, відступами та заголовком.
• Оформіть label, input, select, textarea, fieldset, legend, checkbox, radio і кнопку надсилання.
• Для текстових полів задайте однакову ширину, внутрішні відступи, рамку, заокруглення та читабельний розмір шрифту.
• Додайте щонайменше один стан взаємодії: :hover для кнопок та :focus для полів форми.
• За бажанням використайте :valid, :invalid, :required або :checked для додаткового візуального зворотного зв'язку.
input,
select,
textarea {
 width: 100%;
 padding: 12px 14px;
 border: 1px solid var(--color-border);
 border-radius: 10px;
 background: #fff;
}

input:focus,
select:focus,
textarea:focus {
 outline: none;
 border-color: var(--color-primary);
 box-shadow: 0 0 0 3px rgba(29, 78, 137, 0.15);
}

button {
 padding: 12px 18px;
 border: none;
 border-radius: 10px;
 background: var(--color-primary);
 color: #fff;
 cursor: pointer;
}

button:hover {
 opacity: 0.92;
}
	

5.7. Адаптивність сайту (обов'язково)
• Створіть щонайменше один media query для ширини екрана 768 px або менше.
• На вузьких екранах меню повинно перебудовуватися у вертикальний список або переноситися на кілька рядків.
• Відео, зображення та таблиця не повинні «ламати» ширину сторінки.
• Контейнер, картки та форма мають зменшувати відступи, щоб інтерфейс залишався зручним на планшеті та смартфоні.
• Усі сторінки треба перевірити щонайменше у двох режимах: desktop і mobile.
@media (max-width: 768px) {
 .nav-list {
   flex-direction: column;
   gap: 10px;
 }

 .two-column {
   flex-direction: column;
 }

 h1 {
   font-size: 28px;
 }

 .container {
   width: min(100% - 20px, 1100px);
 }
}
	

5.8. Мінімальний перелік CSS-вимог до виконаної роботи
• зовнішні CSS-файли підключені до всіх трьох сторінок;
• використано щонайменше 10 різних CSS-властивостей;
• використано щонайменше 5 класів для оформлення інтерфейсу;
• використано щонайменше 2 приклади Flexbox;
• оформлено навігацію, таблицю, зображення, відео, аудіо та форму;
• реалізовано щонайменше 1 media query;
• код CSS структуровано блоками та прокоментовано щонайменше в 3 місцях.
5.9. Бонусне завдання (необов'язково)
• Додайте плавні переходи transition для кнопок, посилань або карток.
• Використайте псевдокласи :first-child, :last-child, :nth-child або псевдоелементи ::before / ::after.
• Створіть окремий блок із картками та оформіть його через CSS Grid або складніший Flexbox.
• Реалізуйте темну тему або альтернативну кольорову схему на базі CSS-змінних.

7. Що здати
• Посилання на відкритий репозиторій GitHub або GitLab.
• Повний проєкт із трьома HTML-сторінками та папкою styles.
• Файли style.css і responsive.css з охайно структурованим кодом.
• README.md із коротким описом того, що було стилізовано, та інструкцією запуску.
• Не менше 2 скріншотів: один у desktop-версії, один у mobile-версії.
9. Типові помилки, яких слід уникати
• підключення неправильного шляху до CSS на сторінках у папці pages;
• змішування оформлення в HTML через style="..." із зовнішнім CSS;
• надто яскрава або хаотична палітра кольорів без єдиного стилю;
• відсутність відступів між блоками, через що сторінка виглядає злиплою;
• таблиця або форма виходять за межі екрана на мобільних пристроях;
• використання фіксованих ширин без адаптації;
• відсутність станів hover/focus для навігації та форми;
